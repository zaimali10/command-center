## Round 1 — Kanban Drag & Drop + System Monitor + Theme Toggle + Forecast

### Kanban Drag & Drop (Major)
- **Decision:** Used Pointer Events API (not HTML5 drag-and-drop) for full iPad/mobile compatibility — consistent with existing tap/long-press gesture handling.
- **Gesture design:** 
  - `pointerdown` → track start position, run a short hold timer (120ms)
  - If movement exceeds 10px threshold → enter **drag mode** (card becomes a floating ghost)
  - If movement < threshold → normal tap behavior (advance or long-press edit)
  - On drop → detect which column the ghost overlaps, move card there; also supports drop-order positioning within a column
- **Visual feedback:** Dragged card gets a `kb-dragging` class (shrinks, fades to 0.6 opacity). Drop zones highlight on hover with a glowing dashed border. Drop zone indicator line between cards shows exact insertion point.
- **Reordering:** Cards within a column can be reordered by dropping between cards.
- **Data model:** Added `order` field to cards (timestamp-based) so drop order persists across sessions.

### System Monitor Telemetry (New)
- **Decision:** Python script (`telemetry.py`) using `psutil` — writes JSON to `data/telemetry.json` which serve.py serves as a static file at `/data/telemetry.json`. No extra server process needed.
- **Schedule:** Runs in a background loop every 30 seconds (or via cron for lighter footprint).
- **Data collected:** CPU % (per-core + average), Memory % (used/total), Disk % (used/total), uptime, process count, boot time.
- **Dashboard widget mountMonitor:** Now fetches this JSON instead of showing placeholder. Displays live values with animated bars. Polls every 30 seconds.

### Theme Toggle (New)
- **Decision:** Three themes — **Glass Dark** (default, existing), **Glass Light** (warm white bg, dark text, green accents), **Midnight** (deeper #050a14 bg, blue-tinted accents instead of green/cyan).
- **Implementation:**
  - Theme stored in localStorage (`cc.theme.v1`)
  - Toggle button in header (cycle through themes on tap)
  - Each theme swaps CSS custom properties via `[data-theme="..."]` on `<html>`
  - Theme persisted across sessions

### Weather Forecast (New)
- **Decision:** Added 3-day forecast card on the dashboard. Uses wttr.in API (`/Dallas?format=j1`) — same source as current weather but with the full forecast payload.
- **Display:** Shows day name, condition icon, high/low temps, and precipitation chance for today + next 2 days.
- **Data source:** Combined with existing weather widget (single fetch, shared data).

### 2026-06-02 — Round 1.5: Admin Auto-Start (Zero User Touch After Boot)

**Decision:** Windows requires admin rights for firewall changes. Zaim approved a one-time admin setup rather than manual .bat runs.

**Solution:**
1. `setup-hermes-autostart.bat` — self-elevating script (double-click, approve UAC once):
   - Removes old startup shortcut
   - Creates a **scheduled task** running at logon with **/rl HIGHEST** (silent admin, no UAC)
   - Applies firewall rules immediately
   - Starts Command Center
2. `serve.py` — now detects admin via `ctypes.windll.shell32.IsUserAnAdmin()` and applies firewall rules idempotently
3. Future boots: scheduled task fires silently 30s after logon with full admin privileges

**Trade-off accepted:** One UAC click during setup. Zero interaction ever after.
